在不使用时关闭对象

当使用 Microsoft SQL Server 2005 JDBC 驱动程序的对象时（尤其是 SQLServerResultSet 或 Statement 对象之一，例如 SQLServerStatement、SQLServerPreparedStatement 或 SQLServerCallableStatement），应该在不再需要这些对象时通过使用其 close 方法显式关闭它们。这样可以尽快释放驱动程序和服务器资源，而不必等待 Java 虚拟机垃圾收集器进行此操作，因而提高了性能。

当使用滚动锁定时，关闭对象对维护服务器上良好的并发功能尤其重要。在最后一次访问的提取缓冲区内滚动锁定将一直处于保留状态，直到结果集关闭。类似地，语句预定义句柄将一直处于保留状态，直到语句关闭。如果重用多个语句的连接，则应在其退出作用域之前关闭语句，从而允许服务器清除先前的所有预定义句柄。
