Microsoft SQL Server 2005 JDBC 驱动程序使用 SQL Server 行锁定。这在同时执行数据库修改操作的多名用户之间实现并发控制。默认情况下，以每个连接为基础对事务和锁定进行管理。例如，如果应用程序打开两个 JDBC 连接，则一个连接获得的锁定不能与另一个连接共享。任何连接都无法获得与另一个连接拥有的锁定相冲突的锁定。

注意： 如果使用行锁定，则提取缓冲区中的所有行都会被锁定，因此提取大小设置得过大会影响并发性。

锁定用于确保事务的完整性和数据库的一致性。锁定可防止用户读取其他用户正在更改的数据，并防止多名用户同时更改相同的数据。如果未使用锁定，则数据库中的数据可能发生逻辑错误，并且对这些数据运行的查询可能产生意外结果。

注意： 有关 SQL Server 中行锁定的详细信息，请参阅 SQL Server 联机丛书中的“数据库引擎 中的锁定”。
