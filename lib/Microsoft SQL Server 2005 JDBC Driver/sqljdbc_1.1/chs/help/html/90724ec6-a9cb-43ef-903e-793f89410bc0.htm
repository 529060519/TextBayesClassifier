为了提高 Microsoft SQL Server 2005 JDBC 驱动程序应用程序的安全性，不仅需要避免常见的代码缺陷，还要考虑其他事项。访问数据的应用程序有许多潜在的故障点，可被攻击者利用来检索、操作或破坏敏感数据。从应用程序设计阶段期间威胁建模的进程，到其最终部署及继续完成正在进行的维护，了解安全性的所有方面非常重要。

此部分中的主题介绍了一些常见的安全性考虑事项，包括连接字符串、验证用户输入和一般应用程序安全性。
