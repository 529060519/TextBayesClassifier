构造访问数据的应用程序时，在用户输入得到验证之前，应该假设所有这些输入都是恶意的。如果做不到这一点，您的应用程序就容易受到攻击。可能发生的一种攻击类型称为 SQL 注入，即向字符串添加恶意代码，随后这些字符串被传递给 SQL Server 的实例以进行分析和运行。为避免这类攻击，应尽可能使用带有参数的存储过程，并且总是验证用户的输入。

在客户端代码中验证用户输入很重要，这样不会浪费与服务器的往返通信。在服务器上验证存储过程的参数同样也很重要，可以找出无效输入和未经客户端验证的输入。

有关 SQL 注入以及如何避免其发生的详细信息，请参阅 SQL Server 2005 联机丛书中的“SQL 注入”。有关验证存储过程参数的详细信息，请参阅“存储过程 (数据库引擎)”以及 SQL Server 2005 联机丛书中的附属主题。
