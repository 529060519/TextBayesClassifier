对于所有必须保证其永久性数据的一致性的应用程序而言，事务处理是必需要求。通过 Microsoft SQL Server 2005 JDBC Driver，可以在本地或以分散的方式执行事务处理。事务是原子级的、一致的、孤立的和持久的 (ACID) 执行模型。

本节中的主题介绍 JDBC 驱动程序如何支持包括隔离级别、事务保存点和结果集保持能力在内的事务。
