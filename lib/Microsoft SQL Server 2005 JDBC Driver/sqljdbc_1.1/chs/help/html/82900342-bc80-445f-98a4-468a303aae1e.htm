处理事务时，应尽可能地使事务简短，这一点很重要。默认的自动提交模式将为您提交每项操作，可通过使用 setAutoCommit 方法启用或禁用它。对于大多数开发人员来说，这是最简单的使用模式。

使用手动事务时，请确保您所用的代码能尽可能快地提交事务。使事务处于打开状态会阻止其他用户对数据进行访问。例如，在 catch 块中置入回滚调用，在 finally 块中置入提交调用可能是一种比较有效的编程方案。但是，这取决于应用程序的设计。

使事务尽可能地小会带来更好的并发功能。例如，如果启动手动事务，并修改了行数为 20,000 的表中的 10,000 行，则会导致其他所有用户都无法访问其中的一半部分，即便他们正在读取数据。将修改减少到 2,000 行会使 90% 的表处于可用状态。

此外，如果应用程序会遇到某些阻止问题，且需要对这些问题设置超时时间，请确保使用锁定超时设置。可通过使用 setLockTimeout 方法执行此操作。默认的锁定超时为 0，这表示在等待锁定时将无限期地阻止访问。可将锁定超时设置为 30 秒，如果受其他连接所阻止，这会导致阻止的连接在 30 秒后超时。
