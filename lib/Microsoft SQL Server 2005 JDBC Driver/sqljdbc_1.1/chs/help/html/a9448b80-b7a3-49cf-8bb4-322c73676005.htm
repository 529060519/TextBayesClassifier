以下各节说明 Microsoft SQL Server 2005 JDBC 驱动程序如何为 J2EE 和 JDBC 3.0 的可选 API 功能提供支持。帮助系统中提供的源代码示例为如何开始使用这些功能提供了参考。

首先，请确保 Java 环境（JDK 和 JRE）包含 javax.sql 程序包。任何使用可选 API 的 JDBC 应用程序都需要该程序包。JDK 1.4 及其更高版本已包含该程序包，因此无需单独安装。

驱动程序名称

驱动程序类名称是 com.microsoft.sqlserver.jdbc.SQLServerDriver。该驱动程序包含在 sqljdbc.jar 文件中。

只要加载带有 JDBC DriverManager 类的驱动程序时就需要使用该类名称。必须在驱动程序配置中指定该驱动程序的类名称时也需要使用该类名称。例如，在 J2EE 应用程序服务器中配置数据源时可能需要输入驱动程序类名称。

数据源

JDBC 驱动程序提供对 J2EE / JDBC 3.0 数据源的支持。JDBC 驱动程序 SQLServerXADataSource 类由 com.microsoft.sqlserver.jdbc.SQLServerXADataSource 实现。

数据源名称

可使用数据源连接数据库。下表说明可随 JDBC 驱动程序使用的数据源：

数据源类型 类名称 说明

DataSource

com.microsoft.sqlserver.jdbc.SQLServerDataSource

非池数据源

ConnectionPoolDataSource

com.microsoft.sqlserver.jdbc.SQLServerConnectionPoolDataSource

配置 J2EE 应用程序服务器连接池的数据源。应用程序在 J2EE 应用程序服务器中运行时通常会用到该数据源。

XADataSource

com.microsoft.sqlserver.jdbc.SQLServerXADataSource

配置 J2EE XA 数据源的数据源。应用程序在 J2EE 应用程序服务器和 XA 事务管理器中运行时通常会用到该数据源。

数据源属性

所有数据源都支持设置和获取与基础驱动程序的属性集相关联的任何属性的功能。

示例：

setServerName("localhost");

setDatabaseName("AdventureWorks");

下列内容显示了应用程序如何使用数据源进行连接：

initialize JNDI ..
Context ctx = new InitialContext(System.getProperties());
...
DataSource ds = (DataSource) ctx.lookup("MyDataSource");
Connection c = ds.getConnection("user", "pwd");

有关数据源属性的详细信息，请参阅设置数据源属性。
